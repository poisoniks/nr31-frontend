# OpenAPI Type Integration & DTO Policy
- **Type Source:** Always use the types generated by `openapi-typescript` located in `src/api/types.ts`;
- **No Custom DTOs:** Do not create custom interfaces or classes for Request/Response objects. If a model is missing, it must be updated in the OpenAPI schema of backend and then here, not manually created in the frontend;
- Use indexed access types to reference specific schemas, for example: `type User = components['schemas']['UserDTO'];
- For API functions, explicitly type the return value using the generated response types: `paths['/api/v1/auth/login']['post']['requestBody']['content']['application/json']`;
- **Strictness:** Use `Pick<...>` or `Omit<...>` on generated types if a subset of data is needed for a specific component.
